# OpenCode Go Usage Meter

[English](README.md)

OpenCode Go の利用量に、日割りの目安を追加する Chrome 拡張です。

OpenCode Go のダッシュボードには 5h / 週間 / 月間の使用率とリセット時間が表示されますが、「今日どこまで使ってよいか」は分かりにくいままです。この拡張は既存の Chrome ログインセッションでダッシュボードを読み取り、月間利用量の許容ライン、ツールバーバッジ、しきい値通知を追加します。

## 特長

- OpenCode Go の月間利用量に「今日の残り許容」または「今日の許容超過」を表示
- サイト本来の月間利用量バーに、今日時点の許容ラインを縦線で表示（可能な場合）
- ツールバーのバッジに月間使用率を表示
- 5h ローリング / 週間ローリング / 月間の状態を popup で確認
- 月間の許容超過を 1 日 1 回だけ通知
- 5h ローリング / 週間ローリングが 50%, 75%, 100% に達したとき、リセットまで 1 回だけ通知
- トークン、Cookie、API キーの貼り付け不要

## 対応ページ

- `https://opencode.ai/workspace/<workspace-id>/go`

ワークスペース ID は、OpenCode Go のページを開いたときに URL から自動検出されます。必要な場合は popup から手動設定できます。

## インストール（開発者モード）

1. このリポジトリを clone または zip でダウンロードします。
2. Chrome で `chrome://extensions` を開きます。
3. 右上の **Developer mode** を有効にします。
4. **Load unpacked** を押し、このフォルダを選択します。
5. `https://opencode.ai/workspace/<workspace-id>/go` を開きます。
6. ツールバーの拡張ボタンをクリックすると popup が開きます。

## 使い方

### ダッシュボード上の表示

月間利用量のリセット表示の下に、次のような短いメッセージを追加します。

```text
今日の許容超過: 0.7%（1/30日目・許容 3.3%）
```

今日までの利用ペースを 30 日で日割りして、今日時点で何%まで許容できるかを表示します。

### popup

popup では次を確認できます。

- 5h ローリング利用量
- 週間利用量
- 月間利用量
- リセットまでの時間
- 日割りの許容ライン
- 現在のペースが許容内かどうか

### 通知

通知は Chrome が起動していて、保存済みの workspace ID と有効な OpenCode ログインセッションがある場合に動作します。OpenCode のタブを常に開いている必要はありません。

通知タイミング:

- 月間利用量が今日の許容量を超えたとき: 1 日 1 回
- 5h ローリングが 50%, 75%, 100% に到達したとき: 各しきい値につきリセットまで 1 回
- 週間ローリングが 50%, 75%, 100% に到達したとき: 各しきい値につきリセットまで 1 回
- ログインセッション切れを検出したとき

## 許容ラインの考え方

月間表示と月間通知では、今日までを 30 日で日割りして、今日時点で許容できる使用率を表示します。5h / 週間の popup 表示は、それぞれのリセット時間までの経過ペースをもとに表示します。

## プライバシー

- 通信先は `opencode.ai` のみです。
- トークン、Cookie、API キーは保存しません。
- Chrome の既存ログインセッションを `credentials: 'include'` で利用します。
- workspace ID は URL から検出するか手動入力され、`chrome.storage.local` にのみ保存されます。
- OpenCode 側のデータを変更しません。
- 利用量データを第三者サービスへ送信しません。

## 制限

OpenCode Go には現時点で公開利用量 API がないため、この拡張はダッシュボード HTML に埋め込まれた SolidJS hydration データを読み取ります。サイト側の HTML 形状が変わると、パーサーの更新が必要になる可能性があります。
